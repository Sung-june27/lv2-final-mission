# 회의실 예약 시스템

## 기능 요구 사항

### 예약하기

- [ ] 사용자는 회의실을 예약할 수 있다.

**예외 사항**
- [ ] 공휴일에 예약을 한다면, 예외를 발생한다.
- [ ] 예약이 이미 존재한다면, 예외를 발생한다.

### 예약 조회

- [ ] 모든 사용자는 예약 현황을 조회할 수 있다.
- [ ] 본인의 예약 상세 정보를 확인할 수 있다.
    

### 예약 수정

- [ ] 본인의 예약을 수정할 수 있다.
- [ ] 본인의 예약을 삭제할 수 있다.

**예외 사항**
- [ ] 타인의 예약을 수정하려고 한다면, 예외를 발생한다.
- [ ] 타인의 예약을 삭제하려고 한다면, 예외를 발생한다.

## 도메인 설계

### 멤버(User)

- 이름
- 이메일
- 비밀번호

### 회의실(Room)

- 회의실 이름

### 예약(Reservation)

- 날짜
- 시간
- 멤버 (Entity)
- 회의실 (Entity)
